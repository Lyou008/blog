## 🎓 技术提升路径

### 第 1 周：代码质量意识

**学习目标**: 学会识别代码问题，建立质量意识

**具体行动**:
1. **阅读代码健康报告** (CODE_HEALTH_REPORT.md)
   - 理解什么是"好代码" vs "坏代码"
   - 讨论每个问题的危害和影响

2. **代码审查实践**
   - 每个人 review 他人的代码
   - 使用检查清单：
     - ✅ 有没有明显的 bug？
     - ✅ 有没有安全隐患？
     - ✅ 命名是否清晰？
     - ✅ 有没有重复代码？

3. **工具引入**
   ```bash
   # 安装代码质量工具
   pip install black flake8 pylint mypy bandit
   
   # 格式化代码
   black .
   
   # 检查代码风格
   flake8 --max-line-length=120
   
   # 类型检查
   mypy app.py
   
   # 安全扫描
   bandit -r .
   ```

**验收标准**: 团队成员能独立发现代码中的常见问题

---

### 第 2 周：测试驱动开发 (TDD)

**学习目标**: 理解测试的重要性，学会写单元测试

**具体行动**:
1. **学习测试基础**
   - 什么是单元测试、集成测试、端到端测试
   - pytest 框架使用

2. **为博客项目添加测试**
   ```python
   # test_app.py - 示例
   import pytest
   from app import app
   
   @pytest.fixture
   def client():
       app.config['TESTING'] = True
       with app.test_client() as client:
           yield client
   
   def test_index_loads(client):
       """测试首页能正常访问"""
       response = client.get('/')
       assert response.status_code == 200
   
   def test_login_wrong_password(client):
       """测试错误密码不能登录"""
       response = client.post('/login', data={
           'username': 'admin',
           'password': 'wrong'
       })
       assert b'用户名或密码错误' in response.data
   ```

3. **测试覆盖率目标**
   - 核心功能覆盖率 > 70%
   - 使用 `pytest-cov` 检查覆盖率

**验收标准**: 团队成员能为自己的代码编写测试用例

---

### 第 3 周：架构设计入门

**学习目标**: 理解架构的重要性，学会简单的设计模式

**具体行动**:
1. **学习设计模式**
   - 阅读 `db_unified.py` - 理解适配器模式
   - 讨论：为什么需要抽象基类？
   - 讨论：工厂模式有什么好处？

2. **重构实践**
   - 将项目中的双数据库层合并为一个统一的层
   - 使用 `db_unified.py` 作为参考
   - 确保功能不丢失

3. **Code Review 架构层面**
   - 新功能是否破坏了现有架构？
   - 有没有更好的实现方式？

**验收标准**: 团队能设计出简单的模块化架构

---

### 第 4 周：安全与性能优化

**学习目标**: 理解常见安全漏洞，学会基础性能优化

**具体行动**:
1. **安全审计**
   - SQL 注入防护（我们已经用了参数化查询 ✅）
   - XSS 防护（Flask 自动转义 ✅）
   - CSRF 防护（需要添加 ❌）
   - 密钥管理（需要修复 🟡）

2. **性能优化**
   - 数据库 N+1 查询问题（我们的代码有这个问题 ❌）
   - 使用 JOIN 优化查询
   - 添加数据库索引

3. **部署安全**
   - 不要将 `.env` 提交到 Git
   - 使用环境变量管理敏感信息
   - 生产环境关闭 Debug 模式

**验收标准**: 项目通过基础安全审计，无严重漏洞

## 📋 代码审查检查清单

每次代码审查时，检查以下项目：

### 功能正确性
- [ ] 代码是否实现了需求？
- [ ] 有没有明显的 bug？
- [ ] 边界条件是否处理？

### 代码质量
- [ ] 命名是否清晰易懂？
- [ ] 有没有重复代码？
- [ ] 函数是否太长（> 50 行）？
- [ ] 有没有注释解释复杂逻辑？

### 安全性
- [ ] 有没有 SQL 注入风险？
- [ ] 有没有 XSS 风险？
- [ ] 敏感信息是否泄露？

### 性能
- [ ] 有没有 N+1 查询问题？
- [ ] 有没有不必要的数据库查询？
- [ ] 有没有内存泄漏风险？

### 测试
- [ ] 有没有对应的测试用例？
- [ ] 测试是否覆盖了核心逻辑？
- [ ] 测试是否容易理解？
